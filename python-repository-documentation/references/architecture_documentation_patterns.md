# Architecture Documentation Patterns

## Layer Contract Notes

- Для каждого слоя фиксируй его входные/выходные контракты и зависимость от интерфейсов.
- Для domain слоя подчеркивай изоляцию от инфраструктуры.
- Для application слоя фиксируй CQRS-разделение команд и запросов.

## Aggregate Rules

- Указывай инварианты состояния.
- Отмечай, кто имеет право менять приватные поля агрегата.
- Фиксируй правила версии агрегата и ожидаемые ошибки конфликтов.

## Repository Notes

- Документируй стратегию save и batch-save.
- Явно указывай, что ошибки БД не маскируются.
- Для несерийных ID (UUID и др.) указывай генерацию в коде приложения.

## What to Avoid

- Не описывать очевидную механику языка Python.
- Не переносить в docstring историю изменений и TODO-листы.
- Не смешивать user-facing документацию и внутренние архитектурные заметки в одном блоке.
