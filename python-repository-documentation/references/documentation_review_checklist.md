# Documentation Review Checklist

## Contract Accuracy

- Docstring описывает фактическое поведение текущей реализации.
- Нет утверждений, которые расходятся с тестами.

## Brevity and Value

- Текст короткий и несет новую информацию.
- Нет дублирования аннотаций типов и очевидных имен.

## Section Discipline

- `Args` есть только при неочевидных параметрах.
- `Raises` есть только при контрактно важных исключениях.
- `Returns` есть только при неочевидном результате.

## Layer Rules

- Публичные сущности ключевых слоев задокументированы.
- Приватные методы не перегружены лишней документацией.

## README Quality

- README можно прочитать за пару минут или быстрее.
- Есть разделы: назначение, запуск, конфигурация, тесты, структура.
- Нет избыточных деталей, которые лучше вынести в отдельные файлы.

## Examples Quality

- `env.example` отражает актуальные переменные окружения.
- Примеры конфигов запускаются в local/dev сценарии.
- Секреты и реальные ключи отсутствуют.

## Change Hygiene

- Изменения контракта сопровождаются обновлением docstring.
- Изменения конфигов и env сопровождаются обновлением README и example-файлов.
- Нет устаревших примеров и терминов.
