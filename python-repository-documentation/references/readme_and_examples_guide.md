# README and Examples Guide

## README Target Size

- Пиши README так, чтобы его можно было прочитать за 2 минуты или быстрее.
- Держи фокус на ключевых действиях: что это, как запустить, как настроить, как проверить.

## Recommended README Structure

1. Краткое описание сервиса (2-4 строки).
2. Быстрый старт (установка зависимостей, запуск).
3. Конфигурация (ссылка на `env.example`, ключевые переменные).
4. Команды разработки (линт, тесты, миграции, запуск).
5. Краткая структура проекта (основные директории/слои).
6. Ссылки на расширенную документацию (если нужна).

## env.example Rules

- Включай только переменные, реально используемые приложением.
- Используй безопасные placeholder-значения.
- Добавляй короткий комментарий к неочевидным переменным.
- Не добавляй секреты, токены и реальные DSN.

## Config Examples Rules

- Примеры конфигов должны быть минимальными и рабочими для local/dev.
- Имена параметров и формат должны совпадать с текущей реализацией.
- При изменении схемы конфигурации обновляй примеры в том же изменении.

## What to Avoid

- Длинные теоретические разделы в README без практической пользы.
- Дублирование одного и того же текста между README и другими документами.
- Нерабочие примеры конфигов или устаревшие команды запуска.
