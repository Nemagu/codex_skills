# Pytest Best Practices

## Arrange-Act-Assert

- Структурируй тест по блокам Arrange, Act, Assert.
- Держи один поведенческий контракт на тест, чтобы падение было диагностируемым.

## Parametrize Pattern

```python
import pytest

@pytest.mark.parametrize(
    "raw,expected",
    [
        ("  Alice ", "Alice"),
        ("Bob", "Bob"),
        ("", ""),
    ],
    ids=["trim-spaces", "already-clean", "empty"],
)
def test_normalize_name(raw: str, expected: str) -> None:
    assert normalize_name(raw) == expected
```

## Fixtures

- Фикстура возвращает готовый объект, а не выполняет лишнюю бизнес-логику.
- Избегай autouse-фикстур без необходимости; они скрывают зависимости теста. Исключение — поперечные операции вроде очистки БД перед каждым интеграционным тестом.
- Используй `scope` осознанно: `function` по умолчанию, расширяй только при измеримой пользе. Поднятие тяжёлой инфраструктуры — `session`.

## Markers

- Регистрируй кастомные маркеры в `pyproject.toml`, чтобы было удобно включить `--strict-markers`, когда команда к этому готова:

  ```toml
  [tool.pytest.ini_options]
  markers = [
      "integration: тест требует поднятой внешней инфраструктуры",
      "slow: тест выполняется дольше пары секунд",
  ]
  ```

- Маркируй интеграционные тесты `@pytest.mark.integration`. Маркер используем для **включения** (`uv run pytest -m integration`), а не для исключения — иначе придётся писать `-m "not integration"` со значением через пробел и кавычками, что неудобно в скриптах.
- Чтобы получить быстрый цикл без интеграционных тестов, запускай по директории: `uv run pytest src/tests/units`. Разделение `units/` vs `integration/` всё равно нужно для навигации; маркер дополняет его, не заменяет.
- Общее правило: не передавай в флаги pytest значения с пробелами. Используй односложные маркеры и точечные пути.

## Anti-Patterns

- Проверка нескольких независимых контрактов в одном тесте.
- Неявные зависимости через глобальное состояние.
- Ассерты по внутренним деталям вместо наблюдаемого поведения.
- Тесты, чувствительные к порядку запуска.
